# Remotixy Django Backend

Django REST backend for Remotixy technician accounts, host install tokens, devices, usage reports, billing plan limits, audit logs, and Google AI settings.

## Local setup

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver 8000
```

## API overview

- `POST /api/auth/register/` creates a technician account and organization.
- `POST /api/auth/token/` returns JWT access and refresh tokens.
- `GET /api/auth/me/` returns the signed-in technician profile.
- `GET /api/plans/` lists plan/package limits.
- `GET|POST /api/install-tokens/` manages technician-generated host install links.
- `POST /api/install-tokens/{id}/revoke/` revokes an install token.
- `POST /api/host/devices/register/` lets a host computer register by install token without an account.
- `GET|POST|PATCH|DELETE /api/devices/` manages organization-scoped devices.
- `GET /api/audit-logs/` lists organization audit events.
- `GET /api/usage-reports/` lists organization usage reports.
- `GET|PATCH /api/google-ai/settings/` manages organization Gemini assistant settings.

## Data model

Only technicians have user accounts. Host computers use short-lived install tokens generated by technicians. All devices, reports, tokens, AI settings, and logs are scoped to the technician's organization.

## PostgreSQL

Set either `DATABASE_URL` or the `POSTGRES_*` variables in `.env`.
